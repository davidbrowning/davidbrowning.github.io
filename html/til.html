Today I learned how to host a webpage using nodejs, a little about the express package, and several additional commands to use within vim. The specific vim commands are:

Today I learned that the reason you can't just boot in to safe mode with f8 is that the boot process is just too fast. Technically, the commands f8 or shift + f8 still work, but there isn't enough time for the operating system to recognize them, thus MS added in a way to reach troubleshooting if the OS failes to load 3(?) times.
I also picked up an incredibly useful powershell function:

I found it on this article by Edwin Sarmiento. It queries all the SQL instances on your network, and returns the host and instance names of the SQL servers it finds.

is the typical command on many linux flavors.
You then compare this hash to the one on the downloads page from which you received the file.
If the two are the same, the check passes.

12/19/2016


Today I learned how to access pieces of an expression you've already found with a reg-ex
(Specifically with powershell, but according to powershell.org the syntax is the regex standard)
So you've got a string (We're going to stick with the example from powershell.org) "Don Jones".
You'd like to swap first and last names. Here's how you do it: "Don Jones" -replace "([a-z]+)\s([a-z])"+,'$2,$1'
The result: "Jones Dan"

In the instance where I needed this idea, I had the following two expressions, [regex]$begOfName = '(\d)(,)([A-Z])';, and [regex]$endOfName = '([A-Z].?)(,)(\d)';.
Basically, when you export as a .csv from powershell, in compliance with the .csv standard, it puts " (double quotes) around every item. I didn't want that for the project I was working on, I only wanted quotes around the name of the person behold:
$obj_arr | ConvertTo-Csv -NoTypeInformation |%{$_ -replace('"','') -replace($begOfName,',"$3') -replace($endOfName, '$1",$3') ; }|Out-File output.csv -fo -en ascii

In essence: take my objects, convert them to a .csv replace all the double quotes, then add them back in where I need them. (I'm sure there's a better way to do this, but it was sufficient for what I was doing. Also, I realized that the reason a String[] is passed to the main function in Java is for giving the program command line arguments!

1/10/2017


Today I learned some more useful things from the VIM tutorial,
0 is a much more efficient way to move to the beginning of a line.
:dd will delete the entire line.
:dw will delete a word
:c is the 'change' operator so it will take w,e,$ etc as parameters.
:ce will delete the end of the current word and place the user into insert mode
x will delete a character
I still need to practice the 'count' command (for example: 2w will jump two words to the right, and 4dd will delete 4 lines).
Replace is just Ctrl+R

I also learned a bit about the terminal command 'history'
My quest for this knowledge was fueled by the following addition to my pi's .bashrc file
alias please='sudo (history -p !!)'
Basically, if I execute a command in bash, and have insufficient permissions, I can execute a 'please' command.
This will cause bash to execute the previous command (!!) with elevated permissions as history -p < command >
will simply display the command prior to execution this fits perfectly into the bash alias.
